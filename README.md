# BookFlow

## Overview
BookFlow is a web application that allows users to search, manage, and organize books.  
Users can register books they want to read or have finished reading, helping them track their reading progress and manage their personal book collection.

![image alt](https://github.com/Ferb-10/bookflow-23/blob/7ee4e3fd4dbfb045b5388f13484bd302b3915c9e/public/img/bookflow.png)

## Features
- Search books using external API (Open Library)  
- Add books to a “Want to Read” list  
- Move books to a “Finished” list  
- Display book cover, title, and author information  
- Handle missing data (e.g., no cover image)  
- Dynamic rendering of book lists  
- Simple and intuitive UI for managing reading flow  

## Technologies Used
- Node.js  
- Express  
- EJS 
- Open Library API  
- HTML / CSS  
- JavaScript
- PostgreSQL

## What I Learned
- How to integrate external APIs (Open Library) for search functionality  
- Handling inconsistent API responses (e.g., missing data or no results)  
- Managing application state for multiple lists (want to read / finished)  
- Structuring server-side rendering with EJS  
- Improving user experience with conditional rendering and fallback images  
- Designing a simple system to represent user behavior (reading flow)

## Overview
BookFlowは、書籍の検索・管理を行うことができるWebアプリケーションです。  
読みたい本や読了した本を登録・整理することで、自分の読書状況を可視化し、管理することができます。

## Features
- 外部API（Open Library）を利用した書籍検索機能  
- 「読みたい本（Want to Read）」リストへの追加  
- 「読了（Finished）」リストへの移動  
- 書籍のカバー画像・タイトル・著者情報の表示  
- カバー画像がない場合のフォールバック対応  
- 書籍リストの動的表示  
- シンプルで直感的なUI設計  

## Technologies Used
- Node.js  
- Express  
- EJS
- Open Library API  
- HTML / CSS  
- JavaScript
- PostgreSQL

## What I Learned
- Open Library APIを用いた外部API連携の実装  
- APIレスポンスのばらつき（データ欠損や未ヒット）への対応  
- 複数リスト（読みたい・読了）の状態管理  
- EJSを用いたサーバーサイドレンダリングの構築  
- 条件分岐によるUI改善（画像のフォールバックなど）  
- ユーザーの行動（読書フロー）を意識した設計
